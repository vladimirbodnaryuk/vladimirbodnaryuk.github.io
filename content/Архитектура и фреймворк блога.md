## Архитектура блога

### Два репозитория

```
[Private] vault                    [Public] vladimirbodnaryuk.github.io
github.com/vladimirbodnaryuk/vault     github.com/vladimirbodnaryuk/...

├── inbox/      ← быстрые мысли    ├── content/   ← синхронизируется
├── notes/      ← Цеттелькастен        из published/
├── drafts/     ← черновики        └── → GitHub Pages
├── published/  ──── push ────►         vladimirbodnaryuk.github.io
└── assets/
```

---

### Три точки захвата контента

|Откуда|Куда|Как|
|---|---|---|
|Telegram|`vault/inbox/`|Cloudflare Worker → GitHub API|
|Телефон/десктоп|`vault/inbox/`|Obsidian Mobile / Obsidian|
|Голова → клавиатура|`vault/notes/`|Obsidian напрямую|

---

### Жизненный цикл заметки

```
Telegram / Obsidian
       ↓
   inbox/          ← сырая мысль, не разобрана
       ↓ (обработка в Obsidian)
   notes/          ← атомарная заметка, есть ссылки [[на другие]]
       ↓ (когда готово к публикации)
   drafts/         ← черновик поста
       ↓ (финальная версия)
   published/      ← триггер деплоя
       ↓ (GitHub Action, ~9 сек)
   quartz/content/ ← Quartz строит сайт
       ↓ (~40 сек)
   vladimirbodnaryuk.github.io
```

---

### Стек

|Слой|Инструмент|
|---|---|
|Редактор|Obsidian + плагин Git|
|Хранилище|Git (два GitHub-репо)|
|Быстрый захват|Telegram-бот на Cloudflare Workers|
|Синхронизация|GitHub Actions (rsync published → content)|
|Генератор сайта|Quartz 4 (нативно понимает Obsidian)|
|Хостинг|GitHub Pages (бесплатно)|
|Автодеплой|GitHub Actions (push → build → deploy)|

---

### Ключевые принципы

- **Приватное остаётся приватным** — vault закрыт, публикуется только то что в `published/`
- **Минимум трения** — Telegram или Obsidian Mobile, одно действие для захвата мысли
- **Автоматизация** — пуш в `published/` → сайт обновляется сам, без ручных действий
- **Vendor lock-in отсутствует** — всё хранится как `.md`-файлы, работает с любым инструментом

###  Как добавить свойства в уже созданную заметку

Три способа:

**A. Кнопка в редакторе** — в левом верхнем углу открытой заметки кликни на иконку свойств (⊞ или `<>`). Откроется панель свойств, добавляй поля кнопкой `+`.

**B. Вручную в начале файла** — добавь YAML frontmatter в начало:

```
---
title: Моя заметка
date: 2026-03-08
tags: [идеи]
publish: true
---
```

**C. Вставить шаблон** — `Cmd+P` → "Templates: Insert template" → выбери нужный. Свойства из шаблона вставятся в заметку.

После перезапуска Obsidian шаблоны появятся в боковой панели в папке `templates/`.

---

### Публикация через свойство, а не папку

Да, можно использовать `publish: true` в frontmatter вместо перекладывания файлов. Для этого нужно изменить GitHub Actions workflow — вместо `rsync published/` он будет искать файлы с `publish: true` по всему vault.

Хочешь переключиться на этот подход? Это удобнее — заметка остаётся на месте, просто меняешь свойство. Но потребует небольших изменений в workflow.
